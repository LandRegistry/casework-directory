Background

The prototype was usability tested to ensure it could be used to send the B13 out and then a further round to improve the user experience. On the application details page, after a radio button was selected, the caseworker was required to input information. On the initial prototype, this was revealed on the same page, below the radio button when it was selected. After feedback, and to improve accessibility, in the next round this information was displayed on separate pages.

This design also used revised content for the radio buttons. Caseworkers found it easier to navigate and when the information was displayed across several pages and their interpretation of the radio button content was more consistent. However, due to technical constraints of having questions with a parent-child relationship across multiple pages, the original design, and content, were chosen.

Background

The multi-upload file component was tested with the display showing a box into which an attachment could be added by drag and drop or a select files button. Above the box was a table showing what had been uploaded. Although the functionality of this page tested well there were constraints with timescales making it difficult to technically implement the multi-upload component or show the error messages on the page.

With a new single-upload file component being used instead, the caseworker will select a file to upload then once confirmed they are taken to a separate page with the table showing which file has been uploaded. This process will need to be repeated for each additional file to be added. Time constraints also meant that this design was deployed in the notice service without user testing.
